🔄 Alur OAuth 2.0

1

User Klik "Login dengan Google"

Browser diarahkan ke endpoint /accounts/google/login/ milik django-allauth.

2

Redirect ke Google Authorization Server

Allauth membangun URL dengan client_id, scope, dan redirect_uri, lalu redirect ke Google.

3

User Memberi Izin di Google

Google menampilkan halaman consent. Jika disetujui, Google mengirim authorization_code ke server.

4

Server Tukar Code → Access Token

Django menukar code dengan access_token menggunakan client_secret (server-side, aman).

5

Verifikasi Identitas & Buat Sesi

Allauth mengambil profil dari Google API, membuat/login akun Django, lalu membuat session cookie.

6

Otorisasi Berbasis Email

Server cek apakah user.email ada di ALLOWED_MEMBER_EMAILS. Jika ya → bisa kustomisasi. Jika tidak → hanya bisa melihat.
